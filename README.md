# Freshket Data Lab

An interactive trainer I built to learn **pandas, matplotlib and SQL by doing** — on a realistic (synthetic) Thai B2B food-marketplace dataset. Read a question, try it, take a hint, reveal the answer with real output and a business insight.

▶ **Live:** https://pacharadon.github.io/freshfoodb2bsimulationdata/

> ⚠️ All data here is **synthetic** — generated by script and modeled on the *kind* of business Freshket runs. It contains **no real company data**.

---

## Why I built this
I'm Head of Account Management, moving deeper into the data side of Southeast-Asian food-tech (Agoda / Shopee / Freshket). The fastest way to learn is to build, so I designed a hands-on trainer around a business I understand — a B2B food marketplace — and used it to teach myself the moves a working analyst actually makes: load data, filter, group, chart, and connect tables.

## What it teaches — and what I learned
- **pandas** — reading CSVs, selecting columns, filtering, calculated columns, group-by aggregations, pivot tables, average order value.
- **matplotlib** — bar, line, stacked-bar and scatter charts.
- **SQL** — every data question is shown in SQL too, so I can move fluently between the two.
- **Working across tables** — the project uses three dataframes and shows how they connect through shared dimensions (time, region, channel), not row-for-row joins.
- **32 questions** across **Easy / Intermediate / Hard**, each in the DataCamp style: *context → a think-first task → a halfway hint → reveal (code + real output + a business insight).*

## The three dataframes
| dataframe | what it holds | file |
|---|---|---|
| `transactions_df` | order lines — who ordered what, where, how much (978 rows) | `freshketdanny.csv` |
| `mkt_df` | marketing spend & performance by channel per week — spend, CAC, ROAS, new customers (70 rows) | `freshket_mkt.csv` |
| `ops_df` | the workforce — department, role, location, salary (201 employees) | `freshket_ops.csv` |

Verified figures baked into the lessons: blended **CAC ฿2,879**, blended **ROAS 3.9×**, and **~69%** of staff in frontline warehouse / logistics / sourcing — the true shape of a vertically-integrated food distributor.

## Design decisions (how it evolved)
I iterated on this deliberately, not in one shot:
- **Naming** — tables are named for what they hold (`transactions_df`, `mkt_df`, `ops_df`), never a vague `df`, and each question loads its own CSV with `pd.read_csv` so the data flow is explicit.
- **Question format** — I researched how DataCamp and IBM structure their exercises, then matched it: real context, a task that makes you think, and a hint that gets you about halfway rather than handing over the answer.
- **Plain English** — every explanation was rewritten to drop jargon ("boolean mask", "vectorized") for language a beginner actually follows.
- **Color scheme** — a dark Freshket-green theme with a labeled syntax legend so every token reads at a glance: **DataFrame = green · method = blue · module = orange · keyword/SQL = magenta · string = yellow · number = violet.**

## Tech
- **Front end:** a single self-contained `index.html` + `fk_app.js`, with [Chart.js](https://www.chartjs.org/) for the charts — no build step, no framework.
- **Data:** Python + pandas (`python/generate_dataset.py`) generates the synthetic datasets, seeded for reproducibility.
- **Bonus:** `ai_helper.py` turns a plain-English question into SQL and runs it — the idea behind the trainer's "ask in plain English."

## Run it locally
Open `index.html` in any browser (it loads Chart.js and icons from a CDN). To regenerate the data: `python python/generate_dataset.py`.
