# Freshket-Style B2B Food Marketplace — Synthetic Sales Dataset & Analysis

A portfolio data project: a realistic **synthetic** transaction dataset for a Thai B2B
food-supply marketplace (HoReCa buyers — Hotels, Restaurants, Cafés), plus a matching
**pandas + SQL** analysis. Built to demonstrate data modeling, querying, and turning
business questions into numbers for the Southeast-Asian food-tech space.

> ⚠️ All data is **synthetic**, generated by `python/generate_dataset.py`. It is modeled
> on the *type* of business Freshket runs — it contains **no real company data**.

## Why this project
- **Data modeling** — a transaction table at *line-item grain* (one row per product in an order).
- **SQL & pandas** — the same business questions answered both ways (`sql/queries.sql`, `python/analysis.py`).
- **Business analytics** — B2B food-distribution economics: thin margins, HoReCa segments,
  chain vs. independent buyers, account-manager performance, and provincial expansion.

## The dataset — `data/freshketdanny.csv`
- **Grain:** one row per **order line item**.
- **Size:** 978 line items · 221 orders · 102 customers · one Mon–Sun week (11–17 May 2026).
- **Currency:** Thai Baht (THB).
- **Keys:** `(order_id, line_no)` is unique; `order_id` repeats across multi-item orders;
  `customer_id` repeats across a buyer's orders.

| Group | Columns |
|---|---|
| Order | `order_id`, `line_no`, `order_datetime`, `order_date`, `day_of_week`, `channel` |
| Customer | `customer_id`, `customer_name`, `customer_type` (HoReCa), `is_chain`, `is_key_account`, `account_manager` |
| Geography | `province`, `region_zone`, `city_area`, `customer_since`, `is_new_customer` |
| Product line | `sku_id`, `product_name`, `product_category`, `unit`, `quantity`, `unit_price_thb` |
| Economics | `gross_revenue_thb`, `discount_thb`, `promo_code`, `cogs_thb`, `gross_margin_thb`, `gross_margin_pct` |
| Fulfilment | `payment_terms`, `payment_status`, `delivery_status`, `delivery_date` |

## Selected findings (one trading week)
- **978 line items roll up to 221 distinct orders** — the core lesson: `COUNT(*)` ≠ orders;
  use `COUNT(DISTINCT order_id)`.
- **Average Order Value ≈ ฿12,500**, with thin **~12% blended gross margin** (true to food distribution).
- **Provincial expansion is visible:** Bangkok leads, but up-country demand is real — **Phuket** is the clear #2 market, with **Songkhla, Krabi, and Chiang Mai** also sizable, and newly-won accounts skewing outside Greater Bangkok.
- **Chains place more, smaller-AOV orders; independents fewer, larger ones.**


(the uploaded table is named `freshketdanny`).

## AI helper — ask the data in plain English
`python/ai_helper.py` turns a natural-language question into a DuckDB SQL query (via an LLM),
runs it against the dataset, and returns the result — the same plain-English-to-SQL idea Agoda
uses GPT for in production.

```bash
pip install -r requirements.txt
export OPENAI_API_KEY=sk-...
python python/ai_helper.py "Which province had the highest net GMV?"
```
```python
from ai_helper import ask
ask("Average order value for chains vs independents?")
```
The SQL-execution layer (`run_sql`) works on its own; only the LLM step needs an API key.
